AI 常见词入门

API Token Skills Agent RAG Context

一、LLM（大语言模型）

经常听到 LLM 就是豆包啊 GPT 这些，所谓超级话痨学霸。你就把它当成一个读过全世界所有书的人，知识量巨大，什么都能聊。不管是 ChatGPT 还是 Claude，它们本质上都是 LLM。

它能干嘛呢：它是大脑，负责思考和回答。缺的就是不会主动找你，人会主动反馈沟通的，所以后面需要借用 Agent。

二、Prompt（提示词）

这个大家不陌生，前几年玩 MJ，就是抄各种提示词。所谓的魔法。当然现在依然重要，只不过现在可以用自然语言沟通了。

翻成人话就是：以前用百度是“搜关键词”，现在用 AI 是“提需求”。你对 AI 说的每一句话，比如“帮我写个请假条”“把这段话改成小红书风格”，这就叫 Prompt。

想要用得好：需求提得越细，活儿干得越好。这跟甲方给设计师提需求是一个道理。

三、Hallucination（幻觉）

人话翻译：一本正经胡说八道。因为 AI 是靠预测下一个字来回答的，有时候它其实不知道答案，但为了把话接下去，它会编得特自信。

比如你问它“林黛玉倒拔垂杨柳的情节”，它可能真给你编一段出来。这就是“出现幻觉了”。也会编一些没有的历史故事，本质上 LLM 还是靠学习到的知识去猜测。

所以别指望 AI 给的答案能达到 100% 正确，但现在够用了，很多时候已经能达到非常高的可用度。

四、Token

我们经常会看到“你用了多少 Token”，玩 openclaw 的时候尤其会遇到。它的意思其实很简单：AI 的计费单位。

AI 不按“字数”算钱，也不按“时间”算钱，它按 Token 算。你让 AI 做事，AI 也要吃“饭”才能做。

换算上，大概 1000 个 Token 约等于 700 个英文单词，或者 400 到 500 个汉字。你跟它聊得越多，消耗的 Token 越多。

五、API（接口）

人话翻译：不管饭的服务员。

假设你想在自己的 Excel 里直接用 AI 功能，但你又不想自己造一个 AI。这时候你需要 API。它就像个服务员，把你 Excel 里的问题拿去给 AI 大厨（模型），大厨做好了，它再端回来给你。

场景上，凡是那种“一键生成图”的工具软件，背后很多都是在通过 API 偷偷喊 AI 干活。包括你用 lovart、liblibu 这些，也都是在调用别的模型 API 在干活。

比如你做一个玩偶陪伴，像用豆包，就可以买个豆包的 API 作为玩偶陪伴对话。

六、Agent（智能体）

人话翻译：带脑子的全能助理。这是去年最火的词。普通的 AI 是“你问一句，它回一句”。Agent 是“你给个目标，它自己去想办法”。

比如你说：“帮我策划一次去日本的旅行。”普通 AI 会给你写个只有文字的攻略；Agent 会先查机票价格，再看酒店库存，甚至帮你把预订链接都找好。它能拆解任务，还能自己用工具。

七、Skills / Tools（技能 / 工具）

人话翻译：给 AI 装的外挂。原本的 AI 大脑（LLM）是不能联网、不能画画的，只能打字。Skills 就是给它装上“手”和“眼”。

这就是为什么 openclaw 那么强大，因为它有各种 Skills。装个 Search Skill，它就能联网搜新闻了；装个 Python Skill，它就能帮你跑代码分析数据了。

八、RAG（检索增强生成）

有些答案模型本来是没有的，比如你给你男朋友定的规矩，AI 是不知道的。

人话翻译：开卷考试。你要问 AI 你们公司的内部规定，它肯定不知道，因为它没学过。RAG 就是先把你们公司的手册、文档甩给它。你提问的时候，它先去翻书，翻到了再回答你。

这就叫“开卷考试”，不仅准，还不瞎编。

九、OpenClaw（龙虾）

人话翻译：AI 自动化流水线。这也是我最近常提的一个神器。

如果你想拥有一个自己的 Agent，比如你想找参考灵感，它就帮你搜集、整理、打包给你，或者生成 html 网页。你要做海报，它就自己登录你的即梦 AI 账号给你做海报。

十、Context（上下文）

就是“聊天记忆”。你跟豆包聊天，你问她上礼拜你们说了啥，她是不记得的。

为什么 openclaw 能跟你连续聊天？因为它记得你们刚才说了啥。这个“记忆容量”就是 Context。如果记忆容量不够，聊着聊着它就会忘记你名字，或者忘了前文设定。

当然 AI 会变得越来越简单，就像上个月咱们装 openclaw 还需要搞个大半天，现在直接一键部署安装。

但在这个时代，单纯会用工具已经不稀奇了。如果一个人既有极好的品味，又有有趣的想法，再加上 AI 的效率，这种组合才是真的稀缺。

别把自己限制成只会做图的人。你可以做网页、写内容、跑流程、搭系统，很多空白都还等着你去填。
